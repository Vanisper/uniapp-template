# 自定义 TabBar 动画案例设计

## 目标

在现有自定义 TabBar 基础上实现一个可复用的滑动胶囊动画案例。切换标签时，选中指示器在标签之间平滑移动，标签文字同步变色并轻微上浮，同时保持 H5、App 与各类小程序端的基础行为一致。

## 范围

- 保留现有 `Tabbar` 组件的列表字段映射能力
- 根据当前选中项计算指示器的位置与宽度
- 使用 `transform`、`opacity` 和颜色过渡实现动画
- 使用页面路由作为受控选中值，切换仍通过 `uni.switchTab` 完成
- 补充组件行为测试，并执行类型检查、代码检查与 H5 构建
- 在浏览器中验证选中状态、切换动画与常见移动端尺寸

本案例不新增第三方动画库，不实现中央悬浮按钮、液态凹槽或平台专属原生动画，也不改变现有页面业务内容。

## 交互设计

TabBar 保持固定高度和底部布局。每个标签占据相同宽度，选中胶囊位于对应标签的内容区域内。

切换标签时：

1. 指示器使用水平 `translateX` 从旧标签移动到新标签
2. 指示器的透明度在首次获得有效选中项后变为可见
3. 新标签文字切换为激活色，并向上移动少量距离
4. 旧标签文字恢复默认色和原始位置

动画时长为 260ms，缓动使用平滑的 ease-out 曲线。动画只使用合成友好的属性，避免在切换过程中改变标签栏尺寸。

## 组件边界

### `Tabbar/index.vue`

职责：渲染标签、派生当前索引、展示动画指示器并发出切换事件。

输入保持现有 `list`、`defaultValue`、`valueField`、`textField`、`height`、`color` 与 `activeColor`。为避免把动画实现泄漏给布局层，指示器位置由组件内部根据列表长度和当前索引派生。

输出继续使用 `change` 事件，事件载荷保持 `{ value, text }` 与原始列表项，避免影响两个现有布局调用方。

### `layouts/default.vue` 与 `layouts/tabbar.vue`

职责：提供当前页面路由、标签列表和导航行为。布局层不维护重复的活动索引，也不参与动画计算。

### 页面配置

继续以 `pages.json` 生成的 `tabBar.list` 作为标签数据源。本案例无需扩展图标字段，避免让展示效果依赖额外资源。

## 状态与数据流

`currentPage.route` 作为活动标签的唯一事实来源，经 `defaultValue` 传入 `Tabbar`。组件通过 `valueField` 在 `list` 中查找当前项并派生 `currentIndex`。

用户点击标签时，组件只发出 `change` 事件。布局收到事件后调用 `go(value, true)`，最终由 `uni.switchTab` 更新页面路由；路由变化再驱动活动标签和指示器位置更新。

点击已经激活的标签时不重复发出切换事件，避免无意义导航和动画重启。

## 边界与降级

- `list` 为空时不渲染指示器，也不产生点击事件
- 找不到 `defaultValue` 时回退到首项，与现有组件行为保持一致
- 单个标签时指示器保持在首项，不产生无效位移
- 标签文字过长时使用单行截断，避免撑开等分布局
- 动画 CSS 不依赖滤镜、遮罩、SVG 或浏览器专属 API
- 不支持动态效果偏好的平台仍可正常切换；H5 端通过 `prefers-reduced-motion` 关闭过渡

## 视觉样式

底栏使用克制的中性背景和顶部细分隔线，活动胶囊使用主题激活色的浅色背景。文本字号与容器高度匹配，不使用大号展示文字。胶囊圆角不超过 8px，避免与项目其余工具型界面产生风格冲突。

安全区仍由现有布局体系负责，本组件不重复计算底部安全距离。

## 测试与验收

组件测试覆盖：

- 字符串值能映射到正确活动项和指示器索引
- 数字默认值按索引选择标签
- 无效默认值回退到首项
- 点击非活动项发出一次结构正确的 `change` 事件
- 点击活动项不重复发出事件
- 空列表不渲染活动指示器

工程验证包括：

- `pnpm type-check`
- `pnpm lint`
- `pnpm build`
- H5 桌面和移动视口下的浏览器截图与交互检查

验收标准：切换两个现有 tab 时胶囊位置正确、动画连续、文字状态同步，页面跳转正常；标签栏高度和页面布局不发生跳动；类型检查、代码检查和构建通过。
